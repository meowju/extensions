# V3.7 Architecture: Skill Self-Crystallization

**Based on:** GenericAgent (lsdefine/GenericAgent, arXiv:2604.17091)

---

## 🏗️ System Overview

```
[Task Execution]
      ↓
[Done Hook Trigger] → [HookContext created]
      ↓
[DoneHooksManager] → [SkillCrystallizer.analyze()]
      ↓
[Pattern Detected? (2+ similar)] → No → Skip
      ↓ Yes
[SOP Generated] → [SOPWriter.write()]
      ↓
[FTS5 Indexed] → [Skill stored]
      ↓
[Future Task] → [SkillRecall.query()] → [SOP suggested]
```

## 🔑 Core Interfaces

### DoneHookContext
```typescript
interface DoneHookContext {
  taskId: string;
  taskDescription: string;
  success: boolean;
  toolCalls: ToolCall[];
  duration: number;
  timestamp: Date;
  sessionId: string;
}
```

### Skill SOP Format
```markdown
# skill: {name}

## Trigger
- {when to use}

## Context
- {prerequisites}

## Steps
1. {step 1}
2. {step 2}

## Verification
- {how to verify success}
```

## 🔄 Workflow

### Crystallization Trigger
1. Task completes successfully (DoneHook fires)
2. Tool call sequence extracted (≥2 calls required)
3. Pattern matcher checks for repetition (2+ occurrences)
4. If repeated → generate SOP candidate

### Skill Generation
1. Tool calls → natural language steps (via LLM)
2. Trigger conditions inferred from task context
3. Verification steps derived from success criteria
4. SOP written to `~/.meow/skills/{skill-name}.md`

### Skill Recall
1. New task arrives
2. Query FTS5 with task keywords
3. Return top-3 matching skills
4. If confidence > 0.8 → auto-suggest
5. If accepted → prepend skill steps to execution

## 🛡️ Safety Guards

| Guard | Implementation |
|-------|----------------|
| Duplicate prevention | Check FTS5 before write |
| Dangerous ops | Skip crystallization for shell/git hooks |
| Minimum quality | Require ≥2 tool calls + success |
| User override | All skills marked as "auto-generated" |
