=== Research: Cursor Agent Architecture ===

== What It Does ==
Cursor is a comprehensive AI-first IDE that has evolved from a VS Code fork into a standalone agentic development environment. As of Cursor 3.x, it provides:

- **Agents Window**: A dedicated workspace for running multiple agents in parallel with tiled layout support, persistence across sessions, and cross-repository capability
- **Canvases**: Agent-created interactive visualizations using React components (tables, charts, diagrams, diffs, todo lists)
- **Cursor CLI**: Terminal-based agent workflow with commands like `/debug`, `/btw`, `/config`, `/statusline`
- **Design Mode**: Browser annotation system for precise UI feedback targeting
- **Bugbot**: PR review agent with self-learning rules that adapt from user feedback

== How It Works ==

### Agents Window Architecture
- Split-view architecture allowing multiple agent sessions side-by-side or in grid
- Agents can run locally, in worktrees, in cloud, or via SSH
- Session state persists across restarts
- File tabs scoped to current agent's visible tabs (not globally unique)
- Extension events scoped per workspace to reduce cross-project interference

### Canvas Rendering System
```
Agent decides task benefits from visual/interactive view
    → Agent builds canvas using React-based component library
    → Canvas inserted as reference in chat
    → User can open/reopen canvas, switch to source to edit
    → Canvas saved to workspace for later rerun with fresh data
```
Key components: tables, boxes, diagrams, charts, diffs, to-do lists
Skills can package canvas workflows for consistent outputs

### CLI Workflow
- `/debug`: Hypothesize, add logs, use runtime info to pinpoint issues
- `/btw`: Side question without derailing main task
- `/config`: Interactive settings panel
- `/statusline`: Custom status bar (mode, branch, environment, hints)
- Ctrl+Z to send CLI to background
- Footer shows: working directory, worktree, current branch

### Error Handling Patterns
- Auto-run detection: If user enables auto-run 3x in a row, prompt to turn it on permanently
- Diff-to-file navigation: Jump from diff straight to exact line
- Branch selection in empty state: Prevents accidental wrong-branch runs
- Scoped extension events per workspace

== What Meow Should Steal ==

### 1. Canvas/Artifact System
Agents can create interactive visualizations - Meow could support "cards" or "views" for complex outputs. When agent produces table data, offer to render as interactive view.

### 2. Side Conversation (/btw Pattern)
Allow `/btw` style quick questions that don't interrupt main task. This is a key UX pattern - maintaining context while asking clarifying questions.

### 3. Tiled Agent Layout
Support multiple concurrent agent sessions in split view. Let users compare outputs or work on parallel tasks without tab switching.

### 4. Diff-to-File Navigation  
From diff output, enable jumping directly to exact file:line. This bridges agent output and editor.

### 5. Scoped Tab Management
Tab names resolved within current session's visible tabs, not globally unique. Reduces naming conflicts.

### 6. Session Persistence
Agent state (tiled layout, active conversations) persists across restarts.

### 7. Proactive Detection Patterns
- Detect repeated user actions (like auto-run enablement) and prompt to automate
- Branch selection before agent launch to prevent wrong-target runs

### 8. Voice Input UI
Waveform, timer, cancel/confirm buttons during recording. Full clip → batch STT for accuracy.

## What Meow Should Avoid

### 1. Over-engineering Canvas System
Cursor's React component library for canvases is complex. Meow should start simpler - maybe just structured cards/tables before attempting full visualization.

### 2. Separate Agent Window (VSCode Pattern)
Cursor 3 separated Agents Window from editor. For terminal-first Meow, keep agents in terminal but add ability to detach views.

### 3. Too Many Slash Commands
Cursor's `/config`, `/debug`, `/btw`, `/statusline`, `/update-cli-config` - risk of command proliferation. Keep Meow's slash commands minimal and discoverable.

### 4. Assuming Desktop Environment
Cursor is desktop-only. Meow should stay terminal-first but could offer optional GUI components.

### 5. Implicit Auto-Run Behavior
Cursor had approval loops. Don't assume users want auto-run; explicit confirmation for file changes is safer.

## Next Steps ==

### High Priority
1. **Implement /btw-style side questions** - Allow clarification without task interruption
2. **Add diff-to-file navigation** - `F` key to jump from diff to file:line
3. **Scoped tab resolution** - Tab names within session, not global
4. **Session persistence** - Persist layout and active conversations

### Medium Priority  
5. **Canvas/View cards** - When agent returns structured data, offer interactive view
6. **Tiled sessions** - Support multiple agent sessions in split layout
7. **Proactive detection** - Detect repeated patterns and offer automation

### Lower Priority
8. Voice input UI (waveform, controls)
9. Background/foreground process management
10. MCP server integration

### Key Insight from Iteration 3 Fixes
Cursor's changelog (Apr 13, 2026) explicitly mentions:
> "Fixed a bug where an agent conversation full of diffs or code blocks would flash and freeze"

This was a key Meow finding in Iteration 3! The issue is NOT Meow-specific - it's common across agentic systems. Key fix patterns Cursor used:
- Limited local diff fetches to reduce CPU/network spikes
- Cut dropped frames by ~87% for large edits
- Avoid expensive updates unless truly needed

For Meow: Virtual list rendering and diff limiting are the correct approach.
