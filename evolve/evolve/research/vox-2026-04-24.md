=== Research: Vox Human-Agent Collaboration UI ===

Date: 2026-04-24

## What It Does

Vox is a UI/UX pattern system for human-agent collaboration. It focuses on the interaction layer - how humans and AI agents communicate, collaborate, and share control.

Key concepts:
- Presence indicators (is agent working, thinking, stuck?)
- Incremental transparency (show agent reasoning without overwhelming)
- Natural interruption points (human can interject at appropriate moments)
- Graceful authority transfer (who's in control when)

## How It Works

**Presence & Status**
- Agent states: Idle, Processing, Thinking, Waiting, Blocked
- Visual indicators for each state (not just spinner)
- Progress context: "Indexing 234 files" vs "Processing..."

**Transparency Patterns**
- Show reasoning incrementally, not all at once
- Use collapsible sections for deep reasoning
- Distinguish between "what I'm doing" and "what I found"

**Interruption Model**
- Humans can interrupt at natural boundaries
- Agent checkpoints: "About to delete 3 files, continue?"
- Clear "pause/resume" semantics

**Authority Gradients**
- Fully autonomous (agent decides and acts)
- Semi-autonomous (agent proposes, human approves)
- Supervised (agent suggests, human executes)
- Adjustable per-task or globally

## What Meow Should Steal

1. **Rich Agent States** - Beyond "thinking", show: "Indexing codebase", "Executing command", "Waiting for permission"

2. **Incremental Reasoning Display** - Show agent thinking in collapsible sections. Don't dump all reasoning at once.

3. **Checkpoint Prompts** - Before destructive operations, explicit checkpoint: "This will delete X. Continue?"

4. **Authority Mode Toggle** - Let user choose: /auto (full autonomy), /semi (propose then ask), /help (suggest only)

5. **Progress Context** - "Writing file 3 of 12" is better than "Working..."

6. **Stuck Detection** - If agent is waiting >30s, show "Agent stuck?" prompt

## What Meow Should Avoid

1. **Overwhelming Transparency** - Don't show every token of reasoning. Summarize.

2. **Constant Interruption** - Checkpoints should be meaningful, not every step.

3. **All-or-Nothing Autonomy** - Don't force user to choose between "AI does everything" and "AI does nothing".

4. **Silent Failures** - If agent is stuck, tell the user. Don't just time out.

## Next Steps

1. Add rich agent state indicators to relay output
2. Implement checkpoint prompts for destructive operations
3. Add /auto, /semi, /help authority modes
4. Add progress context to long operations
5. Add "stuck" detection and user prompt
